UI Structure for Simulation
GLOBAL UI ARCHITECTURE
The frontend must be built as a three-mode system with a shared underlying layout framework (React + WebGL/Three.js or Mapbox GL JS). Each mode reuses consistent UI regions (a primary visualization canvas in the center, a context panel on the right side, a control/navigation dock, and a system feed at the bottom) but changes the level of abstraction.
MODE 1: GLOBAL OCEAN INTELLIGENCE OVERVIEW
Purpose: High-level continuous understanding of the ocean system.
Canvas: A continuous fluid simulation of the ocean. No grid lines or discrete dot-based representations. Alkalinity and CO2 are smooth gradient fields; ocean movement uses continuous flow vectors/streamlines.
Context Panel: Aggregated system summaries (global health score, active interventions, high-risk regions).
System Feed: Real-time intelligence updates (anomalies, system alerts).
MODE 2: MISSION CONTROL EXECUTION MODE
Purpose: Support active decision making and simulation of OAE interventions.
Canvas: The global environment compresses to focus only on a specific mission region. Displays a localized ocean field snapshot. Simulates intervention effects over time (dispersion of alkalinity). Uncertainty is visualized using spatial probability shading.
Context Panel: Displays "Mission Cards" generated by the AI agents. Cards include geographic location, dosage, an AI-generated rationale, predicted CO2 uptake, and a risk/confidence score. Includes controls to execute or reject the mission.
System Feed: Mission-critical telemetry (environmental changes impacting dispersion).
MODE 3: ROUTE AND INTERVENTION PLANNING MODE
Purpose: Detailed operational planning for individual ship missions.
Canvas: Focuses on a single mission corridor. The ocean is abstracted; the route is a continuous trajectory line with intervention points embedded. Shows near-field dispersion modeling along the route.
Context Panel: Step-by-step ship instructions, dosage amounts per waypoint, timing constraints, and risk flags.
System Feed: Localized operational alerts (weather changes, ship warnings).
